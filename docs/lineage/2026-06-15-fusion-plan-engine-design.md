# `/fusion` — многомодельный движок планирования

- **Дата:** 2026-06-15 (**v2** — после fusion-саморевью)
- **Статус:** **pending transport validation** — до сборки нужен повторный спайк исправленного транспорта + baseline-A/B (см. Спайки)
- **Тип:** Claude Code skill / plugin (мета-тулинг, отдельный репозиторий `~/code/fusion`)

## История ревизий

- **v1 → v2.** Спека прогнана через саму fusion-методу: Claude + Codex (gpt-5.5, xhigh) + DeepSeek V4 Pro независимо отревьюили её. Три семейства **независимо** дали три одних и тех же BLOCKER'а → высокая уверенность, что они реальны:
  1. Claude был судьёй своих же конкурентов (конфликт роли участник/судья/синтезатор).
  2. Не было операциональной метрики «лучше» → тезис нефальсифицируем.
  3. Vet кроет `file:line`, но не кривое **рассуждение** — главную угрозу.
- Архитектура переработана: независимые агенты на сыром брифе, кросс-верификация («идиот-тест»), консенсус-или-блок, обязательный мульти-угол-челлендж, drop verification-gates.

## Проблема и цель

Инструмент делает не «нормальный», а максимально проработанный план: исследование проблематики, перебор решений, ограничения, сравнение, спайки рисковых допущений, интервью оператора на взвешенных развилках. Выход — план-документ. Реализация — вне скоупа (передаётся в forge / `improve execute`).

Тезис: «fusion beats frontier» (OpenRouter) — ансамбль **разных семейств** обгоняет одну топ-модель, потому что у разных лайнеджей разные слепые зоны. **Тезис пока не доказан для планирования** — первый спайк его проверяет (baseline-A/B), до этого статус «pending».

## Не-цели (YAGNI)

- Роутер «одна модель» — это не fusion.
- Отдельная модель-судья — роль распределена (кросс-верификация + консенсус).
- Персистентная БД дебатов — файлы рядом с планом.
- Авто-применение плана к коду.
- **Verification-gates** (improve-формат) — fusion не исполняет, значит гейты никто не валидирует → мёртвый груз и ложная проверяемость. Заменены на ranked assumptions + operator-unknowns.
- **Отдельная фаза «Better/Simpler» троением** — обязательный мульти-угол-челлендж внутри цикла уже покрывает «проще / не решать».

## Контекст: чем отличается от shadcn/improve

improve режет модели **вертикально** (умная планирует → дешёвые исполняют, субагенты одного семейства). fusion режет **горизонтально** (Claude + GPT + DeepSeek дебатируют сам план, разные лайнеджи). Дополняют друг друга: improve-«умный план → дешёвое исполнение» относится к фазе реализации, которая здесь за скобками. Берём у improve: recon глотает intent-доки (ADR/PRD/CONTEXT.md), self-contained-формат плана, drift-check git-коммита.

## Архитектура: цикл планирования

**Принципы:**

- Каждый агент **независим** — не пересказывает выводы Claude. Получает **сырой бриф** (структура репо, ключевые файлы, ADR, intent-доки), НЕ Claude-аннотацию — иначе слепые зоны Claude становятся общими и тезис разнообразия ломается. Бриф ≤30–40% контекста участника, остальное — на собственный research.
- Каждый агент **челленджит задачу с нескольких самостоятельно выбранных углов**, не планирует слепо. Обязан рассмотреть: задачу можно вообще **не решать** (или не решать при условиях X); можно решить **сильно проще**; решение **зависит от будущих планов**; разные предположения и **разные сценарии развития** — первый выбранный вектор не всегда устойчив.

**Цикл (раунд R):**

```
1. Draft        каждый агент НЕЗАВИСИМО строит/правит полный план (мульти-угол)
2. Cross-verify «идиот-тест»: каждый план проверяет ДРУГАЯ модель
                (Claude→Codex-план, Codex→DeepSeek, DeepSeek→Claude)
3. Aggregate    фидбэк кросс-верификаторов + все планы → общий пакет
4. Re-discuss   все видят всё; челленджат свои и чужие; явно склоняются к варианту
                или поднимают противоречия
5. Consensus-ГЕЙТ  ничего не идёт вперёд без согласия ВСЕХ по material-осям
                (architecture/approach/key-assumptions). Большинство НЕ перебивает несогласного.
                material-split жив → ещё раунд + спайк; cap → оператор разбивает тай;
                оператор недоступен → BLOCKED: no-consensus (не выдуманное согласие)
6. Synthesize   ТОЛЬКО после согласия всех; Claude МЕХАНИЧЕСКИ собирает консенсусный
                документ (выбор — консенсус, не Claude): план + ranked assumptions + operator-unknowns
```

**Кросс-верификатор («тест на дебила», шаг 2):** исходит из того, что автор плана некомпетентен и ему нельзя доверять ничего критичного и некритичного. Перепроверяет **каждое** предположение, утверждение, источник, довод, вывод, каждый поворот мысли. Спорное утверждение проверяется **инструментально** (спайк / запрос доки / grep-контрпример), а не только перечитыванием — это и есть reasoning-level vet, которого требовали все три голоса.

**Конвергенция = повод проверить ПОЧЕМУ**, а не авто-стоп: если три семейства сошлись, подтвердить, что это от реального research (цитируют код, вышли за пределы промпта), а не поверхностное согласие на инлайн-тексте или все втроём ошибаются.

**Адаптивная глубина:** раунды идут, пока критичное не сойдётся; жёсткий cap раундов (бюджет-гейт); `--depth lite|full` — оверрайд.

## Формат плана (plan-only)

- **Self-contained:** точные пути, выдержки кода, конвенции репо с образцом.
- **Assumptions ranked by confidence** — вместо фейковых verification-gates.
- **Operator-unknowns:** явный список того, что агенты не смогли определить сами и что требует человека.
- **Hard boundaries + STOP-условия** — для будущего исполнителя (forge/improve).
- **Drift-check:** штамп git-коммита, против которого писался план.

## Транспорт и адаптеры (заземлено спайком 2026-06-15)

| Участник | Вызов | Роль |
|---|---|---|
| Claude | Claude Code субагент (Agent tool), без записи в репо | оркестратор + участник + кросс-верификатор |
| Codex/GPT | `codex exec --sandbox read-only "<prompt>"`; **фолбэк** `openai/gpt-*` через OpenRouter при исчерпании ChatGPT-квоты | участник + кросс-верификатор |
| DeepSeek **V4 Pro** | `opencode run --dir <repo> -m opencode-go/deepseek-v4-pro "<prompt>"` (контекст ~1M → полный харнесс влезает) | участник + кросс-верификатор |

**Research-харнесс (конкретный лёгкий список, не «уточним потом»):** `grep`, `read` (лимит строк), `glob`, веб-поиск (1 инструмент), read-only `bash` (ls/git log/tree). **Без** write/edit/task-субагентов.

**Подтверждено спайком:** `codex exec` (gpt-5.5) работает неинтерактивно (квота — реальный риск, отсюда фолбэк); `opencode run` поддерживает `-m/--agent/--dir/--pure/-f` (флаг `--file` жадный — сообщение ставить ДО `-f`). Контексты по models.dev: `deepseek-v4-pro` ~1M, `deepseek-r1` 160k. **Build-vs-buy:** `openrouter/fusion` — готовый хостед-мультимодель; чёрный ящик без репо/спайков/интервью, можно подключить опц. голосом.

**Fail-closed:** меньше 2 разных семейств доступно → запуск помечается `degraded: claude-only / two-model`, не выдаётся за полноценный fusion.

## Изоляция, worktree, spike-контракт

- Планирование (драфт/верификация/обсуждение) — **read-only**.
- Спайки (инструментальная проверка спорных допущений) — каждый в throwaway `git worktree` от HEAD.
- **`fusion.sh cleanup --all` в начале каждого запуска** — `trap` не ловит SIGKILL / падение родителя, сироты-worktree подчищаются на старте.
- **Spike-контракт:** hypothesis, max files/time, allowed commands, no new deps by default, evidence required. **Проваленный спайк блокирует зависимую ветку плана** (а не молча игнорируется).

## Интервью оператора

Claude зовёт `AskUserQuestion` когда: спор о принципиальном не сходится после R2; необратимое/дорогое/бизнес-решение; вещь, которую агенты явно отметили как «не можем определить сами». Не дёргать по мелочам.

## Бюджет / ROI-гейт

Перед запуском — лимиты: max rounds (cap), max tokens, max cost, max wall-clock. При превышении — откат в `improve quick`. **Открытый вопрос (блокирующий широкое использование):** прирост против improve не доказан — первый спайк это и проверяет.

## Режимы отказа

- CLI упал/таймаут → участник выбывает, при <2 семействах → `degraded:`.
- Спайк упал → допущение «не подтверждено» + блок зависимой ветки плана.
- Cleanup сирот — `cleanup --all` на старте.

